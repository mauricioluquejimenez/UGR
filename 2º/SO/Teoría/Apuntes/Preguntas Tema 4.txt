1. - Archivo: información relacionada entre sí almacenada en un dispositivo de almacenamiento secundario bajo el mismo nombre. Suele almacenar metadatos como propietario, permisos, hora de última modificación y tamaño.
  - Directorio: elemento que puede agrupar distintos archivos bajo un nuevo nombre. Suele almacenar metadatos parecidos a un archivo (propietario, permisos, hora de última modificación, tamaño), además de un mecanismo para acceder al resto de información

2. Separar nombres (en el directorio) de metadatos (en i-nodos u otra estructura) permite que las operaciones que solo necesitan el nombre y la referencia sean más ligeras: el directorio mantiene entradas compactas y no arrastra todos los atributos en cada consulta. Eso escala mejor cuando hay muchos archivos en un mismo directorio. Sin embargo, cuando necesitas atributos completos (tipo, permisos, tiempos, tamaño), tienes que seguir la referencia y leer el i-nodo correspondiente, lo que puede añadir E/S adicional. Por eso algunas operaciones de “listado detallado” son más costosas que un listado simple de nombres.

3. - En almacenamiento contiguo, los bloques de un archivo están seguidos: eso da acceso secuencial muy rápido y también acceso directo (aleatorio) eficiente porque la asociación lógico→físico es trivial. El problema es que reservar espacio sin conocer el tamaño es complicado y aparece fragmentación externa, a veces con necesidad de compactación.

   - En almacenamiento enlazado, cada archivo es una lista enlazada de bloques: evita fragmentación externa y permite crecer dinámicamente, además puedes leer secuencialmente desde un puntero al primer bloque. El gran coste es que el acceso directo es ineficiente (tienes que recorrer enlaces) y los punteros ocupan espacio. En almacenamiento indexado, un bloque índice guarda punteros a bloques de datos: el acceso directo es eficiente y no hay fragmentación externa pero hay sobrecarga del índice y problemas si el bloque índice es demasiado grande o demasiado pequeño.

4. Para desplazarse dentro de un archivo, el sistema utiliza enlaces entre el inodo y los distintos bloques físicos en los que está almacenado el archivo. Estos enlaces pueden ser directos (enlazan directamente el i-nodo con el bloque físico) o indirectos (entre el i-nodo y el bloque de datos hay una serie de bloques de punteros que apuntan a los datos). 

5. Porque la asignación enlazada, a diferencia de la asignación contigua o la indexada, tiene una traza que se puede seguir y a partir de la cual se puede recuperar información. La asignación contigua simplemente asigna un espacio compacto, pero no alinea de ninguna otra manera los bloques, mientras que el almacenamiento indexado depende del bloque origen pero no establece una secuencia de bloques, cosa que sí hace el almacenamiento enlazado.

6. Las diapositivas describen bitmap (cada bloque tiene asociado un bit: si está libre el bit vale 0; si está ocupado vale 1) y listas que enlazan bloques libres (de manera que si se escribe el primer bloque libre disponible se puede apuntar automáticamente al siguiente bloque libre), que a su vez tienen variaciones como listas enlazadas con agrupación (cada nodo describe un conjunto de bloques libres) y la lista de bloques contiguos, donde se guardan pares (bloque inicial, contador de adyacentes) para representar grandes huecos con muy pocos números.

7. Un sistema de archivos es la parte del sistema operativo encargada de la gestión y el almacenamiento permanente de información. Tiene que crear estructuras y jerarquías (en general, abstracción de archivo, directorio, etc.) para relacionar archivos y directorios con sus respectivos propietarios, permisos, subdirectorios, operaciones disponibles para su gestión (creación, consulta, borrado...) etc.

8. Una partición es la división lógica de un dispositivo de almacenamiento, de manera que el SO trata cada partición como un disco independiente. Esto permite, por ejemplo, tener dos sistemas operativos instalados de forma simultánea. Por otra parte, en el inicio del disco se guarda la información sobre particiones y el arranque: históricamente, en MBR estaban en los primeros 512 bytes la tabla de particiones y el código inicial, con limitaciones (por ejemplo particiones principales). Con UEFI se usa GPT, que supera esas limitaciones y define una partición especial ESP donde residen los cargadores de los sistemas operativos. El superbloque sería la estructura con información global del sistema de archivos dentro de una partición concreta.

9. El journaling aporta un registro parcial de las últimas operaciones realizadas, lo que permite una recuperación parcial ante errores imprevistos o un apagado no limpio. En ext3 se mantiene un registro circular de operaciones y eso permite recuperar de forma ordenada transacciones que quedaron a medias, manteniendo la integridad y reduciendo el tiempo de comprobación tras un apagado no limpio.

10. El VFS (sistema de archivos virtual) permite integrar distintos sistemas de archivos con una capa de abstracción y una interfaz común que define el comportamiento de cualquier sistema de archivos, de manera que los programas de usuario puedan usarlos uniforme y indistintamente.
